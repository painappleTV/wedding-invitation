# iPhoneで動作確認する方法

AWSやReactの経験がなくても大丈夫です。以下の順で試してください。

---

## 方法1: 同じWi-Fiで確認（最も簡単・無料）

PCとiPhoneを**同じWi-Fi**に接続して、iPhoneからPCの開発サーバーにアクセスします。

### 手順

**1. PCで開発サーバーを起動**

プロジェクトフォルダ（`wedding-invitation`）で、PowerShellまたはコマンドプロンプトを開き：

```
npm run dev
```

**2. 表示されたURLを確認**

起動すると、次のような表示が出ます：

```
  ➜  Local:   http://localhost:5174/
  ➜  Network: http://192.168.1.xxx:5174/
```

この **Network** のURL（`http://192.168.x.x:5174`）をメモします。  
※ ポート番号は 5173 や 5174 など、環境によって異なる場合があります。表示されている番号を使ってください。

**3. iPhoneでアクセス**

- iPhoneのSafariを開く
- アドレスバーに `http://192.168.x.x:5174` を入力（xxxはあなたのPCのIP、ポートは表示に合わせる）
- 招待ページは `http://192.168.x.x:5174/invite/abc123` で確認できます

### 注意点

- PCとiPhoneは**同じWi-Fi**に接続していること
- PCのファイアウォールでポート5173/5174が許可されていること（ブロックされる場合は許可設定が必要）
- PCの電源が入っている間だけアクセス可能です

## 方法2: ngrokで一時的に公開（外出先からも確認したい場合）

「家の外から」「別のネットワークから」iPhoneで確認したい場合は、ngrokで一時的なURLを発行します。

### 手順

**1. ngrokをインストール**

- https://ngrok.com/ にアクセス
- 無料アカウントを作成
- ダウンロードして解凍（または `choco install ngrok` など）

**2. 開発サーバーを起動**

```
npm run dev
```

**3. 別のターミナルでngrokを起動**

```
ngrok http 5174
```

※ ポート番号は起動時に表示された番号（5173 や 5174 など）に合わせてください。

**4. 表示されたURLをiPhoneで開く**

`https://xxxx.ngrok-free.app` のようなURLが表示されるので、それをiPhoneのSafariで開きます。

## 方法3: 本番のように公開する（Vercel・無料）

「本番に近い形で」「いつでもiPhoneから」確認したい場合は、Vercelにデプロイする方法が簡単です。

### 手順（概要）

1. https://vercel.com でアカウント作成
2. GitHubにプロジェクトをプッシュ（Gitが初めての場合は別途説明が必要）
3. Vercelで「Import Project」→ GitHubリポジトリを選択
4. ビルドコマンド: `npm run build`
5. 出力ディレクトリ: `dist`
6. デプロイ後、発行されたURL（例: `https://wedding-xxx.vercel.app`）をiPhoneで開く

※ API（出欠フォーム）を使う場合は、別途バックエンド（AWS Lambdaなど）のデプロイが必要です。モックデータのみの確認なら、VercelだけでOKです。

## 用語の簡単な説明

| 用語 | 説明 |
|------|------|
| **React** | このサイトのフロントエンド（画面表示）を作るライブラリ |
| **Vite** | 開発サーバーやビルドを担当するツール。`npm run dev` で起動 |
| **npm** | Node.jsのパッケージ管理。`npm run dev` でスクリプト実行 |
| **AWS** | 本番環境として使うクラウドサービス。Lambda、S3、DynamoDBなど |
| **localhost** | 自分のPCを指すアドレス。他の端末からはアクセス不可 |
| **192.168.x.x** | 家庭内LANのIPアドレス。同じWi-Fi内の端末からアクセス可能 |

## トラブルシューティング

**「接続できません」と表示される**
- PCとiPhoneが同じWi-Fiか確認
- PCのファイアウォールで5173/5174を許可
- `npm run dev` が起動したままか確認
- ポート番号（5173 や 5174）が正しいか確認

**「招待が見つかりません」と表示される**
- URLが `/invite/abc123` または `/invite/xyz789` になっているか確認（モックデータはこの2つのみ）

**API（出欠送信）を試したい場合**
- 別ターミナルで `npm run dev:api` も起動する
- 方法1の場合、ViteのプロキシでAPIも転送されるので、そのまま使えます
